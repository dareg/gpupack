About the evolution of the toolbox

GmkPack is regularly upgraded as the code become more and more complex.

This is mostly because the rules to make the executables, and the ODB and blacklisting ''plug-ins'' are embedded inside GmkPack .

However GmkPack is easy to upgrade : the installation is fast ; then one may have only to re-build the compilation scripts. Packs made with an older version is always working with a newer version.
