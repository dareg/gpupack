Executables

Lower in the script you will find such lines : 
# Do you want to make any executable ? (yes/no)
# ----------------------------------
Load=yes
You can switch off the making of all executables by setting Load=no.
Lower in the script you will find such lines for each executable : 
# Binary "aladin" :
cat <<end_of_aladin_link> $GMKWRKDIR/.aladin_link
EXEC=ALADIN
ENTRY=arp/control/master.o
end_of_aladin_link
# BACKGROUND LIBRARIES (from top to bottom) :
cat <<end_of_aladin_libs> $GMKWRKDIR/.aladin_libs
/home/khatib/pack/opt/lib/libald.main.a
/home/khatib/pack/opt/lib/libarp.inter.1.a
/home/khatib/pack/opt/lib/libarp.main.a
/home/khatib/pack/opt/lib/libtal.main.a
/home/khatib/pack/opt/lib/libtfl.main.a
/home/khatib/pack/opt/lib/libxrd.main.a
/home/khatib/pack/opt/lib/libsur.main.a
/home/khatib/pack/opt/lib/libdummysat.main.a
/home/khatib/pack/opt/lib/libdummyodb.main.a
/home/khatib/pack/opt/lib/libdummybla.main.a
/home/khatib/pack/opt/lib/libdummympa.main.a
/home/khatib/pack/opt/lib/libdummymse.main.a
/home/khatib/pack/opt/lib/libunsxref-verbose.main.a
end_of_aladin_libs
# LOW-LEVEL LIBRARIES (from top to bottom) :
cat <<end_of_aladin_sys> $GMKWRKDIR/.aladin_sys
-L$GMAPLIBPATH -lec -lgribex -lfdbdummy -lwamdummy -lnaglitedummy -loasisdummy
-llapack
-lblas
-L$GMAPLIBPATH -lmpidummy -libmdummy
end_of_aladin_sys
# LOADER AND OPTIONS :
cat <<end_of_aladin_load> $GMKWRKDIR/.aladin_load
pgf90
end_of_aladin_load
For each executable you can change for instance :
  • the name given to the executable file (EXEC)
  • the libraries or the order of libraries to be linked (see the parts BACKGROUND LIBRARIES and LOW-LEVEL LIBRARIES)
  • the loader and its options (see LOADER AND OPTIONS)
