Compilation of selected dependencies

When you modify a module or a header file, you should recompile itself and all its dependencies as well. By default GmkPack determines and compile all the dependencies for a given module or header. You can decide not to compile the dependencies, or to compile only a part of them (choosing the projects) : 

# Do you want to compile dependencies ? (yes/no)
# -----------------------------------
export DEP=yes
# If you compile dependencies, approve the following projects list :
export MKPROJECT="sur mpa mse xrd tfl tal arp ald sat odb uti bla"
Notice :
  • Changing the default behaviour is highly unsafe ! This option had been developped in order to run GmkPack faster and before the cycle 28 of ARPEGE/IFS was released. Since this cycle and its numerous explicit interfaces, this option has become quite dangerous. Don't change it unless you are a code expert or you are sure that you did not change any module nor any header nor any subroutine interface.
  • This option does not appear in compilation scripts from main packs, where the whole code should be compiled.
